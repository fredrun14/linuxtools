## Contexte

### Problématique
La sécurité d'injection est OK (commandes via `CommandBuilder` → argv sans
`shell`). Mais `name`/`group_name` ne sont pas validés : un nom commençant par `-`
pourrait être interprété par `useradd`/`groupadd` comme une **option**. De plus,
`ensure_user_groups` réussit silencieusement même si tous les groupes demandés
sont absents.

### Solution technique retenue
Valider le format des noms (réutiliser la regex de `systemd/validators.py` ou en
créer une dédiée) et/ou utiliser le séparateur `--`. Documenter le contrat
best-effort de `ensure_user_groups`.

### Fichiers impactés
- `src/linux_python_utils/identity/user.py`
- `src/linux_python_utils/identity/group.py`
- `tests/test_identity_user.py`, `tests/test_identity_group.py`

## Évolutions à mettre en place (Détail Junior)

### `user.py` / `group.py` — 🟡 MINEUR (validation noms)
```python
import re

_NOM_UNIX = re.compile(r"^[a-z_][a-z0-9_-]*\$?$")  # convention useradd

def _valider_nom(nom: str) -> str:
    if not _NOM_UNIX.match(nom):
        raise ValueError(f"Nom Unix invalide : {nom!r}")
    return nom
```
Appeler `_valider_nom` avant de construire la commande. Alternative : ajouter
`--` comme séparateur d'options dans `CommandBuilder`.

### `user.py:80` — 🟡 MINEUR (contrat)
Documenter dans la docstring que `ensure_user_groups` est **best-effort** : un
groupe inexistant est loggé puis ignoré, et la fonction réussit même si aucun
groupe n'a pu être appliqué. Envisager un retour `list[str]` des groupes
réellement appliqués.

### Gestion d'erreurs
| Cas | Condition | Action |
|---|---|---|
| Nom invalide | regex échoue | `raise ValueError` |
| Groupe absent | `getgrnam` → `KeyError` | log warning + skip (documenté) |
