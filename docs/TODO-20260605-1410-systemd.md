## Contexte

### Problématique
1. **Injection de directives** : seul `Environment=` est protégé contre les `\n`.
   Tous les autres champs sérialisés (`description`, `exec_start`, `options`,
   `what`, `where`...) acceptent un `\n` → injection de directives arbitraires
   dans le fichier unit.
2. **TOCTOU + path traversal** : `unit_porter.restore` (l.298-300) écrit sans
   `O_NOFOLLOW` et avec un `unit_name` non validé (déduit du stem d'un TOML) →
   écriture arbitraire dans `/etc/systemd/system`.
3. `executor.py` ne valide que le radical du nom d'unité (`rsplit(".",1)[0]`).

### Solution technique retenue
- Helper `_reject_control(value, champ)` appliqué à TOUS les champs sérialisés.
- `restore` réutilise le helper TOCTOU-safe de `filesystem` + valide `unit_name`.
- Validation du nom d'unité complet (radical + extension whitelistée).

### Fichiers impactés
- `src/linux_python_utils/systemd/base.py` — filtrage `\n` dans `to_unit_file`
- `src/linux_python_utils/systemd/unit_porter.py` — TOCTOU + validation
- `src/linux_python_utils/systemd/executor.py` — validation complète
- `src/linux_python_utils/systemd/validators.py` — helpers
- `src/linux_python_utils/systemd/config_loaders/` — code mort + bug f-string

### PR1 — SÉCURITÉ (🔴 BLOQUANT)

#### `validators.py` — ajouter les gardes
```python
import re

_EXTENSIONS_VALIDES = {"service", "timer", "mount", "automount", "socket"}

def reject_control_chars(value: str, champ: str) -> str:
    """Rejette les sauts de ligne et caractères de contrôle.

    Raises:
        ValueError: Si value contient \\n, \\r ou un caractère de contrôle.
    """
    if any(ord(c) < 32 for c in value):
        raise ValueError(
            f"Caractère de contrôle interdit dans le champ '{champ}'."
        )
    return value

def validate_full_unit_name(unit_name: str) -> str:
    """Valide nom + extension d'une unité systemd."""
    if "." not in unit_name:
        raise ValueError(f"Nom d'unité sans extension : {unit_name!r}")
    radical, ext = unit_name.rsplit(".", 1)
    if ext not in _EXTENSIONS_VALIDES:
        raise ValueError(f"Extension d'unité non autorisée : {ext!r}")
    validate_unit_name(radical)  # fonction existante (regex + anti-traversal)
    return unit_name
```
