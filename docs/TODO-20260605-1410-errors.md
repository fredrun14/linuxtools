## Contexte

### Problématique
Trois paramètres injectés au constructeur sont **inopérants**, ce qui viole le
principe d'injection (DIP) du projet et rend l'API trompeuse :
- `console_handler.py:46` : `handle` teste `isinstance(error, ApplicationError)`
  en dur, ignorant `self.base_error_type`.
- `console_handler.py:38` : `solutions` est stocké mais jamais consulté (cascade
  `isinstance` codée en dur l.62-79).
- `logger_handler.py:17-26` : `base_error_type` accepté mais jamais stocké.

S'ajoute une fuite de nom projet aval (`FlatpakAutoUpdateError`) dans une lib
générique.

### Solution technique retenue
Utiliser les paramètres injectés (ou les supprimer s'ils n'ont pas de sens). Ici :
les utiliser, car ils font partie du contrat d'injection voulu.

### Fichiers impactés
- `src/linux_python_utils/errors/console_handler.py`
- `src/linux_python_utils/errors/logger_handler.py`
- `tests/test_errors.py`

## Évolutions à mettre en place (Détail Junior)

### `console_handler.py` — 🟠 MAJEUR
1. Routage via le type injecté :
```python
def handle(self, error: Exception) -> None:
    if isinstance(error, self.base_error_type):
        self._handle_known_error(error)
    else:
        self._handle_unknown_error(error)
```
2. Consulter `self.solutions` (dict injecté `type[Exception] -> str`) dans
   `_handle_known_error` au lieu de la cascade `isinstance` en dur :
```python
solution = self.solutions.get(type(error))
if solution:
    print(solution)  # ou via un logger sur stderr
```
3. Router les messages d'erreur vers **stderr** (pas `print` sur stdout).
4. Remplacer la mention `FlatpakAutoUpdateError` par `ApplicationError` (doc).

### `logger_handler.py` — 🟠 MAJEUR
Stocker et utiliser `base_error_type` :
```python
def __init__(self, logger: Logger,
             base_error_type: type[Exception] = ApplicationError) -> None:
    self._logger = logger
    self._base_error_type = base_error_type

def handle(self, error: Exception) -> None:
    if isinstance(error, self._base_error_type):
        self._logger.log_error(f"Erreur connue : {error}")
    else:
        self._logger.log_error(f"Erreur inattendue : {error}")
```
Corriger la docstring « différents niveaux » si seul `log_error` est utilisé.
